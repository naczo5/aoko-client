/ features

core modules and capabilities.

01. combat
autoclicker
lmb/rmb with configurable cps, jitter, block-only mode, and gui/chest safety checks.
aim assist
adjustable fov, targeting range, and smoothing strength.
triggerbot
crosshair-gated firing with cooldown threshold and hit-chance controls.
speedbridge
edge sneak assist with block-only, hold-sneak, looking-down, and safety timing controls.
reach & velocity
extra combat tuning — reach extender and velocity dampener with hit-chance.
autototem
ghost (inventory only) and anarchy modes, health threshold, and elytra switch.
02. visuals
nametags
player tags detailing health, armor tier, and held items.
closest player
on-screen tracking panel displaying direction, exact distance, and equipment.
chest esp
chest highlighting through walls with count limits and distance culling.
module list & logo
configurable in-game module list (default, minimal, outlined, glass, bold) with optional logo.
gtb helper
reads guess-the-build hints and shows candidate words in an overlay list.
discord rich presence
live state, version, and active module count surfaced through discord rpc.

/ architecture

loader and bridge overview.

01. loader
c# / .net 8 wpf
finds javaw.exe and injects the native bridge dll. owns the external gui, profiles, keybinds, and discord rpc.
02. bridge
c++ jni overlay
attaches to jvm/jni, reads game state, and renders imgui overlays through SwapBuffers. config and state sync over tcp 25590.

/ setup

requirements and quick start.

01. requirements

windows 10/11 x64.

lunar client (1.8.9, 1.21.x, 26.1).

release exe is self-contained — no .net install needed.

mingw-w64 + jdk 17 headers (build only).

02. usage
  1. launch lunar client.
  2. run Aoko.exe.
  3. click inject.
  4. use the external gui after inject.
  5. bind keys under the keybinds tab to toggle modules.
